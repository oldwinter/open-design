## 这不是一次 release，而是一声发令枪。

过去十年，designers 每天早上都在做同一件事：打开 Figma、推像素、拖矩形、命名 layers、同步 tokens、交付 spec、看着 engineering 重新解释其中一半、然后重做 spec。有人这样过了五年，有人这样过了十年。**整个职业被变成了单一工具的点击农场。**

我们认为那个时代已经结束。要让它真正结束，必须同时满足两件事，而 0.8.0 是第一次让两者在同一个 engine 中成为现实的 release：

### Movement 1 — **#DeFigma**：杀死 canvas，保留 craft。

Figma 没有坏。它只是 agent-native 世界里*错误的工作单元*。Design system 不该是一个你手工维护的文件。Prototype 不该是 200 个需要手动微调的 frames。Slice export 不该是人类仪式。**Canvas 曾经只是没有真正 engine 时的 workaround。**现在我们有了 engine。

### Movement 2 — **Agent-Native Design**：designer 成为 director。

Claude Design、Open Design — agent-driven design 不再是 demo。它正在交付 production-grade design systems、world-class UI、decks、prototypes，而且服务于那些*不会画矩形*的人。不是五年后。**就是今天，在这个 preview 中。**人们一直承诺 AI tooling 会带来 10×–100× productivity jump？它已经来了，而且不是来自一个强行贴到 Figma 上的 chat box。它来自一个 agent 是 first-class citizen、人类给出 intent 而不是 coordinates 的 engine。

> 每个 designer、每个 team、每家公司，都能交付 world-class design systems、interfaces、decks 和 prototypes。这就是赌注。这就是 0.8.0 被构建出来的原因。

我们还年轻，还粗糙，但已经有 40k stars，并且正在加速。**我们需要你帮我们把剩下的路推完。**

## 为什么我们重建 core（以及它为何对 movement 重要）

0.7 的最后两周说明了一件事：我们最初的 research-preview architecture 承载不了 community 想把它带去的方向。因此我们没有继续打补丁，而是**重建了 foundation**，因为一个 movement 需要的是其他人也能在其上构建的 kernel，而不是只有作者能扩展的 demo。

0.8.0-preview 就是这次重建。底层有三件事不同了：

- **Everything is a plugin.** Engine 保持小而无聊。Design systems、slices、prototypes、exports，*以及包括 Figma 本身在内的每个 legacy workflow*，现在都存在于 plugins 中。**旧世界就是这样被移植进新世界的：一次一个 plugin，由我们所有人完成。**你写一个 plugin，就迁移了行业的一块。
- **Headless by default.** Desktop app 只是 CLI 外的一层薄 wrapper。同一个 engine 可以跑在 Claude Code、OpenClaw、Hermes Agent 中，也可以直接跑在 Lark / Discord / Slack chat 里。**你可以用 Claude Code 驱动它。Claude Code 也可以被它驱动。**Design 不再是一个你要前往的地方，而成为你的 agents 拥有的一种 capability。
- **Plugins create plugins.** OD CLI 包装 GitHub CLI，因此 agent 可以 clone repo、scaffold plugin，并打开 PR，可以为你，*也可以为它自己*。Loop 闭合了：engine 在公开环境中自我生长，而你在 loop 中。

这一切背后的更大赌注是：**一个整个行业都能在其上构建的 agent-driven design engine，open source、vendor-neutral、agent-runtime-neutral。**它不是又一个 AI design tool，而是未来十年 design 所站立的 substrate。

## 我们需要你。就是你。三种参与方式。

这才是重要部分。Movement 不是从某个团队的 laptops 中 ship 的；它来自那些早早出现、并补齐缺失拼图的人。

### 1. 大声把它弄坏。

任何内容都算数：crashes、"this feels off"、"I wish I could…"、令人困惑的 copy、加载慢。**小报告最有用。**不要把半成形的想法憋着；我们想要它。

→ [**Open a 0.8.0 preview feedback issue**](https://github.com/nexu-io/open-design/issues/new?template=preview-v0.8.0-feedback.yml) — bugs、suggestions、first impressions 都欢迎。Team 会关注 [all open issues](https://github.com/nexu-io/open-design/issues)。

对于并非 bug 的开放想法，请**直接回复这个 thread**。每条回复都会被阅读。

### 2. 写一个 plugin。迁移旧世界的一块。

这是你能为 movement 做的最高杠杆事情。Engine 故意保持很小，*因为 plugin surface 才是行业被 port 进来的地方。*如果你的 team 有 Figma workflow、token sync、export pipeline、brand kit、deck template、prototype recipe，请**把它包装成 OD plugin，那个 workflow 会在一夜之间变成 agent-native。**既为你的 team，也为其他所有人。

我们会亲自帮助你 ship 第一个 plugin。打开一个标题为 `Plugin idea: <thing>` 的 issue，或在 Discord 上 ping 我们。

如果你想直接提交 fix，请照常向 `main` 打开 PR；0.8.0 work 现在在那里。[看看当前 open 的内容。](https://github.com/nexu-io/open-design/pulls)

### 3. 告诉一个人。

找一个厌倦推像素的 designer，一个厌倦等待 specs 的 PM，一个厌倦第五次重建同一个 component 的 engineer。把这个 thread 发给他们。**Movement 会以口碑传播的速度扩张。**

## 我们真心想听你怎么看的几件事

我们能猜到 0.8 哪些地方已经稳了。下面这些才是我们猜不到的部分：

1. **Plugin framing.** 当你实际使用时，"everything is a plugin" 是否*感觉*正确？有没有某个地方你想扩展它，但 seam 让人觉得不对？
2. **Headless.** 如果你通过 CLI、Claude Code、OpenClaw、Hermes Agent，或 Lark/Discord agent 试过它，感觉如何？魔法在哪里断掉？
3. **0.7 → 0.8 jump.** 你在 0.7 依赖的东西有没有消失、移动，或行为不同？
4. **Authoring a plugin.** Agent-guided "create a plugin" flow 是否为你端到端跑通，还是在哪里跌倒？
